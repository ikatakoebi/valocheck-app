# 製品仕様書: VALOCHECK — 日本語Valorant戦績トラッカー

## 概要
**コンセプト**: Riot IDを入力するだけで、Valorantの戦績・ランク・マッチ履歴を日本語で確認できるWebサイト
**ターゲット**: 日本語話者のValorantプレイヤー（自分や友人の戦績を手軽に確認したい人）
**コアバリュー**: 日本語で信頼できるValorant戦績トラッカーが存在しないため、そのギャップを埋める

## 機能一覧

### F01: プレイヤー検索 [P0]
**説明**: Riot ID（名前#タグ）を入力してプレイヤーを検索する
**ユーザーストーリー**: プレイヤーとして、Riot IDを入力して自分や他のプレイヤーの情報ページにアクセスしたい。なぜなら戦績を確認する入口が必要だから。
**受け入れ基準**:
- [ ] トップページに名前とタグの入力フォームがある
- [ ] 「名前#タグ」形式で1つのフィールドに入力でき、#で自動分割される
- [ ] 検索ボタンまたはEnterキーで検索が実行される
- [ ] 存在するプレイヤーを検索するとプレイヤーページに遷移する
- [ ] 存在しないプレイヤーの場合、エラーメッセージが表示される

### F02: ランク・ティア表示 [P0]
**説明**: プレイヤーの現在のランク（ティア）、RR（ランクレーティング）を表示する
**ユーザーストーリー**: プレイヤーとして、自分や他人の現在のランクをひと目で確認したい。なぜならランクはValorantにおいて最も重要な指標だから。
**受け入れ基準**:
- [ ] プレイヤーページにランクティア名が日本語で表示される（例: ダイヤモンド1）
- [ ] 現在のRR（0〜100）が数値で表示される
- [ ] ランクに対応するアイコン画像が表示される
- [ ] アンレートのみのプレイヤーは「ランクなし」と表示される

### F03: マッチ履歴一覧 [P0]
**説明**: 直近のマッチ履歴をリスト表示する
**ユーザーストーリー**: プレイヤーとして、最近のマッチ結果を一覧で振り返りたい。なぜなら自分のパフォーマンス傾向を把握したいから。
**受け入れ基準**:
- [ ] プレイヤーページに直近のマッチ（最低5件）がリスト表示される
- [ ] 各マッチに以下の情報が表示される: 使用エージェント名、マップ名、K/D/A、勝敗、スコア（例: 13-7）
- [ ] 勝利は緑系、敗北は赤系で視覚的に区別される
- [ ] マッチをクリックするとマッチ詳細ページに遷移する

### F04: マッチ詳細ページ [P0]
**説明**: 個別のマッチの全プレイヤーのスコアボードを表示する
**ユーザーストーリー**: プレイヤーとして、特定のマッチの全参加者のパフォーマンスを確認したい。なぜなら自分の立ち位置やチームの状況を把握したいから。
**受け入れ基準**:
- [ ] マッチの基本情報が表示される（マップ、モード、スコア、日時）
- [ ] 味方チームと敵チームが分かれて表示される
- [ ] 各プレイヤーに以下が表示される: エージェント名、K/D/A、ACS（平均コンバットスコア）、ヘッドショット率
- [ ] チーム別にラウンド勝利数が表示される

### F05: エージェント統計 [P1]
**説明**: プレイヤーのエージェント別の使用率・勝率を表示する
**ユーザーストーリー**: プレイヤーとして、自分がどのエージェントを多く使い、どれが得意かを知りたい。なぜならエージェント選択の参考にしたいから。
**受け入れ基準**:
- [ ] プレイヤーページにエージェント別の統計セクションがある
- [ ] 使用回数、勝率、平均K/D/Aが表示される
- [ ] 使用回数の多い順にソートされている

### F06: マップ統計 [P1]
**説明**: プレイヤーのマップ別の勝率を表示する
**ユーザーストーリー**: プレイヤーとして、マップごとの得意不得意を把握したい。なぜなら苦手マップの練習を優先したいから。
**受け入れ基準**:
- [ ] プレイヤーページにマップ別の統計セクションがある
- [ ] 各マップの試合数と勝率が表示される
- [ ] 勝率が視覚的にわかりやすく表示される（バーやパーセンテージ）

### F07: 検索履歴 [P1]
**説明**: 最近検索したプレイヤーをローカルに保持し、素早く再検索できる
**ユーザーストーリー**: プレイヤーとして、よく見るプレイヤーをすぐに再検索したい。なぜなら毎回IDを入力するのは面倒だから。
**受け入れ基準**:
- [ ] トップページに最近検索したプレイヤーの一覧が表示される（最大10件）
- [ ] 履歴にはプレイヤー名とランクが表示される
- [ ] 履歴をクリックするとそのプレイヤーページに直接遷移する
- [ ] 履歴はブラウザを閉じても保持される（ローカルストレージ）

### F08: ダークテーマUI [P0]
**説明**: Valorantの雰囲気に合った赤/黒基調のダークテーマ
**ユーザーストーリー**: プレイヤーとして、Valorantの世界観に合ったクールなUIで戦績を見たい。なぜなら体験の一貫性が重要だから。
**受け入れ基準**:
- [ ] 背景は黒〜ダークグレー基調
- [ ] アクセントカラーは赤（Valorantブランドカラー）
- [ ] テキストは白〜ライトグレーで可読性が高い
- [ ] 全体的に統一感のあるデザイン

### F09: レスポンシブ対応 [P1]
**説明**: モバイルデバイスでも快適に使えるレイアウト
**ユーザーストーリー**: プレイヤーとして、スマホからでも戦績を確認したい。なぜなら外出先やゲーム中のロード画面でも見たいから。
**受け入れ基準**:
- [ ] 768px以下の画面幅でもレイアウトが崩れない
- [ ] テーブルが横スクロール可能になるか、カード型に変形する
- [ ] タッチ操作で支障なく使える

### F10: ローディング・エラー状態 [P0]
**説明**: データ取得中のローディング表示とエラーハンドリング
**ユーザーストーリー**: プレイヤーとして、データを待っている間やエラーが起きた時に何が起きているか知りたい。なぜなら無反応なUIは不安になるから。
**受け入れ基準**:
- [ ] API呼び出し中はローディングインジケータが表示される
- [ ] プレイヤーが見つからない場合「プレイヤーが見つかりません」と表示される
- [ ] API制限に達した場合「しばらく待ってから再試行してください」と表示される
- [ ] ネットワークエラー時に適切なエラーメッセージが表示される

### F11: プロフィールヘッダー拡張 [P0]
**説明**: プレイヤーのプロフィールヘッダーにピークランク、アカウントレベル、リージョンフラグを表示する
**ユーザーストーリー**: プレイヤーとして、過去最高ランクやアカウントレベルをひと目で確認したい。なぜならプレイヤーの実力を総合的に判断する材料になるから。
**受け入れ基準**:
- [ ] ピークランク（過去最高ランク+RR+ACT名）が表示される
- [ ] アカウントレベルが数値で表示される
- [ ] リージョン（AP, EU, NA等）が表示される

### F12: Competitive Overview統計 [P0]
**説明**: プレイヤーの通算統計をCompetitive Overviewセクションとして表示する
**ユーザーストーリー**: プレイヤーとして、自分の通算K/D、勝率、ADRなどをひと目で確認したい。なぜなら長期的なパフォーマンス把握が重要だから。
**受け入れ基準**:
- [ ] 以下の通算指標が表示される: Damage/Round, K/D Ratio, Headshot%, Win%, ACS, KAD Ratio, Kills/Round
- [ ] Kills, Deaths, Assistsの通算数値が表示される
- [ ] DDΔ/Round（ダメージ差分/ラウンド）が表示される
- [ ] 各主要指標（Damage/Round, K/D, HS%, Win%）が目立つカードで表示される

### F13: Accuracy（命中精度）セクション [P1]
**説明**: Head/Body/Legs のヒット割合を視覚的に表示する
**ユーザーストーリー**: プレイヤーとして、自分のエイム傾向（頭/体/足の割合）を視覚的に把握したい。なぜならエイム改善の指針になるから。
**受け入れ基準**:
- [ ] Head/Body/Legs のヒット割合がパーセンテージで表示される
- [ ] 各部位のヒット数が表示される
- [ ] 人体のシルエットまたはバーで視覚的に表現される

### F14: ロール別統計 [P1]
**説明**: Duelist/Controller/Initiator/Sentinel の4ロール別に勝率とKDAを表示する
**ユーザーストーリー**: プレイヤーとして、どのロールが得意かを知りたい。なぜならロール選択の参考にしたいから。
**受け入れ基準**:
- [ ] 4ロール（デュエリスト/コントローラー/イニシエーター/センチネル）ごとのセクションがある
- [ ] 各ロールに勝率（W-L数含む）とKDAが表示される
- [ ] ロールアイコンが表示される

### F15: 武器統計 [P1]
**説明**: プレイヤーの武器別キル数と命中率を表示する
**ユーザーストーリー**: プレイヤーとして、どの武器が得意かを知りたい。なぜなら武器選択の参考になるから。
**受け入れ基準**:
- [ ] 上位3〜5武器のキル数が表示される
- [ ] 各武器のHead/Body/Legs割合が表示される
- [ ] 武器名とカテゴリ（Assault Rifles等）が表示される

### F16: エージェント統計テーブル拡張 [P1]
**説明**: エージェント統計テーブルにADR, ACS, DDΔ, Best Mapを追加する
**ユーザーストーリー**: プレイヤーとして、エージェントごとの詳細な性能指標を比較したい。なぜなら自分のプレイスタイルに合うエージェントを見つけたいから。
**受け入れ基準**:
- [ ] エージェント統計にマッチ数、勝率、K/D、ADR、ACS、DDΔが表示される
- [ ] 各エージェントのBest Map（最も勝率が高いマップ+勝率）が表示される
- [ ] エージェントアイコンが表示される

### F17: マッチ履歴一覧の強化 [P0]
**説明**: マッチ履歴を日別グルーピングし、詳細指標とフィルタを追加する
**ユーザーストーリー**: プレイヤーとして、日ごとのパフォーマンスを把握し、特定のエージェントやマップのマッチだけを見たい。なぜなら効率的な振り返りがしたいから。
**受け入れ基準**:
- [ ] マッチが日別にグルーピング表示される
- [ ] 各日のサマリー（W-L, K/D, HS%, ACS）が表示される
- [ ] 各マッチにDDΔ, HS%, ACSが追加表示される
- [ ] 各マッチに順位（MVP, 3rd, 6th等）が表示される
- [ ] 各マッチにマッチ時点のランクアイコンが表示される
- [ ] エージェントアイコンが表示される（名前だけでなく）
- [ ] 経過時間（4時間前、3日前等）が表示される

### F18: マッチ詳細ページ拡張 [P0]
**説明**: マッチ詳細にラウンドタイムライン、追加指標列、Performanceタブ、Economyタブ、Duelsタブを追加する
**ユーザーストーリー**: プレイヤーとして、マッチの詳細な流れ（ラウンド進行、エコノミー推移、対面勝率）を確認したい。なぜなら試合の反省と改善に必要だから。
**受け入れ基準**:
- [ ] マッチヘッダーに試合時間と平均ランクが表示される
- [ ] ラウンドタイムライン（各ラウンドの勝敗+勝利方法アイコン）が表示される
- [ ] スコアボードに以下の列が追加される: +/-(K-D差), DDΔ, ADR, KAST, FK, FD, MK
- [ ] Performanceタブ: 各プレイヤーのラウンドごとのK/D可視化
- [ ] Economyタブ: ラウンドごとのBank/Loadout推移グラフ
- [ ] Duelsタブ: プレイヤー間の対面キル数マトリクス

### F19: プレイリスト・ACTフィルタ [P1]
**説明**: Competitive/Premier/Unrated等のプレイリスト切替と、ACT期間のフィルタ
**ユーザーストーリー**: プレイヤーとして、コンペティティブだけの戦績やACTごとの成績を見たい。なぜなら期間やモード別の分析がしたいから。
**受け入れ基準**:
- [ ] Competitive / Unrated / Team Deathmatch のプレイリスト切替ボタンがある
- [ ] 切替によりマッチ履歴と統計がフィルタリングされる

## UI/UX要件

### 画面一覧
| 画面名 | 目的 | 主要要素 |
|--------|------|----------|
| トップページ | プレイヤー検索の入口 | ロゴ、検索フォーム、検索履歴 |
| プレイヤーページ | 戦績の総合表示 | ランク表示、マッチ履歴リスト、エージェント統計、マップ統計 |
| マッチ詳細ページ | 個別マッチのスコアボード | マッチ情報ヘッダー、チーム別スコアボード |

### 画面遷移
- トップページ → [検索実行] → プレイヤーページ
- プレイヤーページ → [マッチクリック] → マッチ詳細ページ
- マッチ詳細ページ → [プレイヤー名クリック] → そのプレイヤーのプレイヤーページ
- どの画面からでも → [ロゴクリック] → トップページ

### インタラクションパターン
- 検索フォーム: 入力 → Enter/ボタン → ローディング → 結果/エラー
- マッチ履歴: リスト表示 → クリックで詳細遷移
- エラー時: インラインメッセージ表示（ページ遷移しない）

## データモデル（概念）

### プレイヤー
- 識別子: Riot ID（名前 + タグ）、PUUID
- 属性: アカウントレベル、カード画像、リージョン

### ランク
- ティア名、ティア番号、RR
- 所属プレイヤー

### マッチ
- 識別子: マッチID
- 属性: マップ、モード、開始日時、スコア
- 参加者リスト（各チーム）

### マッチ参加者
- プレイヤー参照
- エージェント、チーム
- K/D/A、ACS、ヘッドショット率、ダメージ

## 制約・エッジケース

### API制約
- レート制限: 30リクエスト/分（Basic Key）。キャッシュで対応必須
- 非公式APIのため、予告なく停止・変更の可能性あり
- リージョン（ap/eu/na/kr等）を正しく指定しないとデータが取得できない

### エッジケース
- ランクがないプレイヤー（アンレートのみ）→「ランクなし」表示
- マッチ履歴がないプレイヤー → 「マッチ履歴がありません」表示
- 名前に特殊文字を含むRiot ID → URLエンコード処理
- #を含まない入力 → バリデーションエラー
- 同名のプレイヤーが異なるリージョンに存在する場合 → リージョン選択UIで対応

### セキュリティ
- APIキーはサーバーサイドのみで管理し、クライアントに露出させない
- ユーザー入力のサニタイズ（XSS対策）
